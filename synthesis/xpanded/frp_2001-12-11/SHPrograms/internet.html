Preliminaries

 

What is the difference between Win2000 Professional, Server, & Advanced Server?

            The Windows2000 product line offers three separate operating systems. Professional is intended to replace the familiar NT Workstation. Professional provides many security enhancements and improved internet/intranet-working features.

Server is geared to provide file and print sharing for an intranet. This version of the operating system supports 4 processors and allows for more “internetworking security” (ex. IP Restrictions). Advanced Server includes the same features as Server, but even more targeted for the Internet. Advanced Server was designed to provide services that will make on-line transaction processing (OLTP) and e-commerce more reliable.

Should I upgrade from Win98?

            Probably not. If Win98 is your operating system of choice and provides everything you need for simple home or client use, then Win2000 is not for you. However, users running on the NT 4.0 platform should consider an upgrade.

Does Win2000 Professional provide good Internet services?

            The answer to this question totally depends on what you are looking for. Win200 Professional is intended to be a workstation platform, not a server. Although it offers Internet services, it is not designed to handle the volume of users that Server or Advanced Server are designed to accommodate.

            This presentation was done with Professional, but there are no differences in the setup or menu structure. There are a few instances where Professional doesn’t offer a feature, but an upgrade to Server or Advanced Server will enable them.

Web Services Setup

To install:

            Add/Remove Programs…Add/Remove Windows Components

To manage web services:

            Internet Services Manager (under Administrative Tools)

Creating A Website

            Default Directory is C:\Inetpub\wwwroot

            Right clicking “Default Web Site” allows administrator to specify default permissions and connection limits.

            NOTE: Win2000 Professional has a limited number of connections, whereas Server and Advanced Server do not.

Performance

            Bandwidth Throttling

Allows administrator to limit connection bandwidth dedicated to Internet services.

            Process Throttling

Allows administrator to limit CPU usage for serving web pages and executing scripts.

Home Directory

            Local

·        Define local path to web space

·        Set Permissions

o       Script Source

o       Read

o       Write

o       Directory Browsing

o       Logging

o       Indexing

·        Application Settings

o       Execution Permissions

o       Refer to Applications section

Redirect

·        Allows specification of URL for browsers to be redirected to. View online reference for parameters.

Documents

Define the default documents to be served when accessing a folder. Common examples are index.html, default.html, etc.

Directory Security

            Anonymous Access

·        Enable or Disable

o       Most commonly enabled so users don’t have to log in to see content.

·        Anonymous Login

o       Can be modified to be any login. It is very important to remember that the user specified here will be the login used to retrieve content. It is best to let IIS manage this.

·        Certificates

o       Used for security. Must first obtain certificate.

HTTP Headers

            Useful when needing to add header information to every page.

MIME Map

Associates file types with server applications that execute them. For example, adding Perl script support.

Custom Errors

Contains html source for all server error messages. This section provides a convenient way to view the server error messages and modify the information the user sees when these errors occur.

Server Extensions

This section allows administrator to specify which type of scripting support will be provided on server. Mail specifications may also be made.

Virtual Directories

A virtual directory provides a way to reference a web space that is either local or somewhere else on the network, that is not actually in the pre-defined web space.

Creating

           

Using Internet Information Services Snap-in, select Action…New…Virtual Directory and use the Wizard.

If using NTFS, right-click from explorer and select Sharing in Item properties.

Applications

Windows 2000 can view a defined web space as an “Application”. By default, the web services include several applications used for management and on-line help.

An Application is defined by viewing a home or virtual directory properties and clicking on “Configure”.

This is yet another way to extend the script processing capabilities of Win2000.

Configure

Application Mappings:

·        Executable: The program that will process the script.

·        Extension: The extension on files that will be processed by executable.

Application Options:

            Allows fine tuning of running applications.

Application Protection:

·        Low à Application is executed in IIS process

·        Medium à Pooled execution

·        High à Isolated execution

Security

Here are some basic security precautions that will help ensure a secure web server:

·        Use NTFS – Higher security and more control

·        Review Directory Permissions – Double check defaults. They are not always right.

·        Store Executables in a separate directory

·        Use most secure form of authentication possible

·        Site Certificates!

·        Synchronize Web and NTFS permissions – The most restrictive permissions will always be enforced, however, the Web and NTFS permissions should be the same!

·        Be as restrictive as possible!
