WebLogic Server 5.1

Installation

  • Perform the install from the the BEA WebLogic Evaluation CD.

Configuration

  • Copy the WebLogicLicense.xml file to \WebLogic\License\
  • Set environment variables
    • path=%JAVA_SDK%\bin;%WEBLOGIC%\bin\oci815_8;%WEBLOGIC%\bin

WebLogic Properties

These describe the changes that we have done to the WebLogic properties file to get it all to work.

  • Edit any references to WebLogic or JDK directories
  • Add a connection pool

    #
    # Add a connection pool
    # Substitute your development directory for D:\pics
    #
    weblogic.jdbc.connectionPool.pics=\ url=jdbc:weblogic:oracle,\ driver=weblogic.jdbc.oci.Driver,\ loginDelaySecs=1,\ initialCapacity=4,\ maxCapacity=10,\ capacityIncrement=2,\ allowShrinking=true,\ shrinkPeriodMins=15,\ refreshMinutes=10,\ props=user=pics;password=pics;server=PICS.BEDFORD.FOLIAGE.COM #
    # pics is the user name for the Oracle database.
    # pics.bedford.foliage.com is the TNS
    # service name for Oracle. The Oracle database will require
    # you to run Net8 config
    # for this to work (when the database is first set up)

  • Give people permission to use the database

    weblogic.allow.reserve.weblogic.jdbc.connectionPool.pics=everyone

  • Make it possible to use JDBC to get to the data source

weblogic.jdbc.TXDataSource.weblogic.jdbc.jts.pics=pics

  • To deploy an EJB
    • change the weblogic.ejb.deploy statement

      #
      # PICS
      #
      # Substitute your development directory for D:\pics

      weblogic.ejb.deploy=\
      D:/pics/deployment/nmc2.jar

    • Add the EJB to /weblogic/myserver/serverclasses
  • Copy any NMC2 html (we don't have any, yet) files to the weblogic/myserver/public_html directory.
